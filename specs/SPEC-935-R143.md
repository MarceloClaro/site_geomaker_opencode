# SPEC-935-R143 — Revisão e Refino do Geomaker (site + TouchTerrain + Bridge OpenCode)

## Objetivo

Revisar, refinar e testar o artefato Museu Geomaker: site estático com
TouchTerrain, Ancient Earth, Geólogo Digital (OpenCode bridge), e cadastro
local de acervo — garantindo que a infraestrutura WSL, os serviços systemd,
o proxy Nginx e os endpoints estejam íntegros e testados via SDD/TDD.

## Motivação

O Geomaker é o primeiro *deployment real* do ecossistema OpenCode como
backend de IA conversacional (Geólogo Digital), via bridge FastAPI + CLI.
Até R143, todo o trabalho foi feito sem especificação formal nem testes
programáticos no core do ecossistema — as alterações em `/opt/geomaker/` e
`/home/marceloclaro/Geomaker_site/` estavam órfãs de specs.

Esta frente documenta e testa:
- A **topologia de deploy** (Nginx → TouchTerrain :8081, Nginx → Bridge :8082)
- As **correções de bugs** (botão "Carregar gerador aqui" removido, erro.txt saneado)
- O **Geólogo Digital** como caso real de uso do OpenCode CLI como backend

## Escopo

### Em escopo
- **Infraestrutura**: verificar que os 3 services systemd estão ativos
  (geomaker-touchterrain, geomaker-opencode-bridge, nginx); porta 8080
  responde 200; proxy reverso roteia corretamente `/main`, `/export`,
  `/preview/`, `/download/`, `/static/`, `/touchterrain/`, `/api/`.
- **Site estático**: todas as 11 páginas HTML servidas; JS/CSS sem erros
  de sintaxe; `erro.txt` saneado (zero falsos positivos de código do site).
- **TouchTerrain**: página dedicada (`touchterrain.html`) serve via iframe;
  `/main` carrega sem 500; tradução pt-BR aplicada.
- **Geólogo Digital (Bridge)**: endpoint `POST /api/chat` responde; health
  check `GET /health` retorna 200; modelo gratuito `opencode/deepseek-v4-flash-free`
  é o padrão; CORS habilitado.
- **Acervo local**: CRUD com localStorage funcional; merge com Tainacan
  (demo oculto quando há cadastro local); admin-acervo.html acessível.
- **Botão "Carregar gerador aqui"**: comprovadamente removido do
  `laboratorio.html`.
- **Erro.txt**: análise atestando que 0 erros são do código do site.

### Fora de escopo

## Critérios de Aceitação

1. `systemctl is-active nginx geomaker-touchterrain geomaker-opencode-bridge`
   retorna `active` para os 3 serviços.
2. `curl -s -o /dev/null -w "%{http_code}" http://localhost:8080/` → 200.
3. `curl -s -o /dev/null -w "%{http_code}" http://localhost:8081/main` → 200.
4. `curl -s -o /dev/null -w "%{http_code}" http://localhost:8082/health` → 200.
5. Proxy Nginx: `curl http://localhost:8080/touchterrain/` redireciona para
   TouchTerrain (status 200 ou 302, não 404).
6. Proxy Nginx: `curl http://localhost:8080/api/health` → 200 (Bridge).
7. `grep -c "Carregar gerador" /opt/geomaker/site/laboratorio.html` → 0.
8. `erro.txt` — contagem de linhas de **extensão Chrome** vs **erro do site**:
   100% dos falsos positivos identificados; zero falsos negativos.
9. Endpoint Bridge: `POST /api/chat` com corpo JSON `{"prompt":"teste"}`
   retorna 200 ou 422 (validação), nunca 500.
10. Páginas HTML do site (`index.html`, `acervo.html`, `laboratorio.html`,
    `admin-acervo.html`, `touchterrain.html`) têm sintaxe HTML válida
    (tags fechadas, DOCTYPE presente).
11. `python3 -c "import sys; sys.path.insert(0,'.'); from sdd.spec_engine import SpecRegistry; r=SpecRegistry(); assert len(r.specs)>=40"` —
    ecossistema segue íntegro após adicionar R143.
